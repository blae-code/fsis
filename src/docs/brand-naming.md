# FSIS Brand & Naming Rules

## Canonical Names

| Context | Use |
|---|---|
| Full brand | FairShare Industrial Solutions |
| Abbreviation | FSIS |
| UI uppercase | FAIRSHARE INDUSTRIAL |
| Public surface | Storefront |
| Internal shell | OS Desktop |
| Admin module | Management Console |
| Old/archived term | Operator Terminal |

## Voice

- Buyer-facing copy should be clear, safety-first, and lightly immersive.
- Admin-facing copy can use operational/terminal language.
- Never let lore obscure critical trade/payment instructions.

## Safety Copy

Use direct wording for payment and identity:

- “Pay only in the in-game trade window.”
- “Keep your passphrase private until handoff.”
- “FSIS will confirm availability before fulfillment.”

## Avoid

- Mixing FairShare/Fairshare casing.
- Calling archived features active.
- Advertising shortcuts or modules that are not mounted in the current surface.
